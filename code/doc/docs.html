Audio Modules
Common Behaviour:
[module]
 => 
Closes the module. The module is also destroyed and grabage collected if it is not connected to any input of any other module.

[module : *io]
 => 
Reports names of all inputs and outputs.

[module module]
 => 
Used for test purposes to push the model one sample forward.

[module []]
 => 
Disconnect all inputs.

[module source]
 => 
Connects the first output of source to the first input of module.

[module DESTINATION source]
 => 
Connects the first output of source to the DESTINATION input of module.

[module DESTINATION source SOURCE]
 => 
Connects the SOURCE port of source to the DESTINATION input of module.

[module DESTINATION source []]
 => 
Disconnects the first output of source from the DESTINATION input of module.

[module DESTINATION source SOURCE []]
 => 
Disconnects the SOURCE port of source from the DESTINATION input of module.

Note: both DESTINATION and SOURCE can be either text or number.
[module *x]
 => 
Reports the current value of the first output port of the module.

[module SOURCE *x]
 => 
Reports the current value of the SOURCE output port of the module.

[module value]
 => 
Sets the value of the first input port of the module.

[module DESTINATION value]
 => 
Sets the value of the DESTINATION input port of the module.

Note: value can be either integer or double.
The commands above describe common behaviour of all audio modules.
Every module has a number of inputs and outputs, which can be connected. It should be noted that several output signals can be connected to one input. In such cases all those signals are added together. In fact, this is the simplest method of building a mixer.
[adsr module corrector] => Creates an adsr module. The optional corrector (default = 0.125) specifies attack curve correction.
Note: module adheres to Common Behaviour.
Signals: [["ENTER" "GATEWAY"] ["SIGNAL"]]
ADSR module. It works on a constant slope basis.
INPUTS:
Trigger: key on/off input. Range between 0.0 .. 1.0.
Attack: time. Range between -8192.0 .. 8192.0. ZERO = 1 second (at correction = 0.0). Every step of 768 (6 * 128) multiplies time by 2.
Decay: time. Range same as above.
Sustain: level. Range between -16384.0 .. 0.0.
Release: time. Range same as above.
OUTPUTS:
Signal: output value. Range between -16384.0 .. 0.0.
Busy: 0.0 = if finished release stage, 1.0 = otherwise.
[amplifier module] => Creates an amplifier module.
Note: module adheres to Common Behaviour.
Signals: [["ENTER" "GATEWAY"] ["SIGNAL"]]
This is an exponential audio amplifier. The range of "GATEWAY" control signal is -16384.0 .. 0.0. Every -1536 (-128 * 12) reduces the input signal by half.
[auto module data maximum_change] => Creates a sequencer connected to auto_data (maximum_change is optional).
Note: module adheres to Common Behaviour.
Signals: [["TRIGGER"] ["SIGNAL"]]
Plays back control sequences stored by the connected data module. The control changes can be ramped (filtered) if maximum_change was specified (ZERO is the default value = no ramping). The exact behaviour depends on the value of the "CONTROL" and "TRIGGER" signals of the auto data block.
trigger = 0 or control = 0 => inactive (no playback)
control = 1 => play once only
control = 2 => play looped
control >= 16 => inactive (no playback)
[auto_data module] => Creates a block holiding a sequence of control changes.
Note: module adheres to Common Behaviour.
Additional usage:
[module *value] => Returns the sequence of control changes.
[module []] => Erases all data.
[module [*value1 *time1] [*value2 *time2] ....] => Enters new data.
Otherwise defaults to common behaviour.
Signals: [["ENTER" "TRIGGER" "CONTROL"] []]
Auto data block holds control sequence as pairs of value and sample number after which the next value should be played. The source of the control values is the "ENTER" input of the data block.
A new sequence can be recorded by feeding 16 (or more) to the "CONTROL" input and then starting the recording by feeding 1 to the "TRIGGER" input. The recording stops when triggers disengages. However, recording may be paused by temporarily disengagin the control. Please, note, the recording should be stopped appropriately (by disengaging trigger while control is still 16).
[chorus module] => Creates a chorus module.
Note: module adheres to Common Behaviour.
Signals: [["SIGNAL" "LEVEL" "TIME" "SPEED" "AMP"] ["SIGNAL"]]
Monophonic chorus effect with integrated LFO block.
INPUTS:
Signal: input signal.
Level: mixing. Range between 0.0 .. 16384.0 linear, where 16384.0 represents equal mix.
Time: shift. Range between 0.0 .. 16384.0 linear, where 8192.0 represents 1/10 of a second.
Speed: of LFO oscillation. Range identical as in lfo. Amp: level of LFO oscillaction. Range 0.0 .. 16384.0 linear, where 16384.0 represents variation equals to two time shifts.
OUTPUTS:
Signal: output.
[control module] => Creates a control module.
Note: module adheres to Common Behaviour.
Signals: [["ENTER" "GATEWAY"] ["SIGNAL"]]
Multiplies two signals. Good for building linear amplifiers and applying sensitivity to control signals.
[delay module] => Creates a dealy module.
Note: module adheres to Common Behaviour.
Signals: [["LEFT" "RIGHT" "FEEDBACK" "TIME" "HIGHDAMP"] ["LEFT" "RIGHT"]]
Stereo cross delay module.
INPUTS:
Left and Right: stereo inputs.
Feedback: level. Range between 0.0 .. 16384.0.
Time: delay. Linear range between 0.0 .. 16384.0, where one second = 8192.0.
Highdamp: filter. Range between 0.0 .. 1.0 (default = 1.0, means no filtering).
OUTPUTS:
Left and Right: stereo output.
[eg module] => Creates a 4-stage envelope generator module.
Note: module adheres to Common Behaviour.
Signals: [["TRIGGER" "TIME1" "TIME2" "TIME3" "TIME4" "LEVEL1" "LEVEL2" "LEVEL3" "LEVEL4"] ["SIGNAL" "BUSY"]]
Envelope Generator module. It works on a constant slope basis. I.e. the times are based on transitions across 16384.0 change between levels. As such, smaller differences between levels will cause the times to get shortened.
Note: if retriggered before reaching level 4, it may start from a point between levels 4, 1 or 2.
INPUTS:
Trigger: key on/off input. Range between 0.0 .. 1.0.
Time 1 - 4: range between -8192.0 .. 8192.0. ZERO = 1 second (at correction = 0.0). Every step of 768 (6 * 128) multiplies time by 2.
Level 1 - 4: range between -8192.0 .. 8192.0 (however, can be more really).
OUTPUTS:
Signal: output value. Range between -16384.0 .. 0.0.
Busy: 0.0 = if reached level 4, 1.0 = otherwise.
[gateway module] => Creates a gateway module.
Note: module adheres to Common Behaviour.
Signals: [["ENTER" "GATEWAY"] ["SIGNAL"]]
Works similar to control module, but its "SIGNAL" output is divided by 16384. Good for building linear amplifiers and applying sensitivity to control signals.
[impulse module] => Creates an impulse module.
Note: module adheres to Common Behaviour.
Signals: [["ENTER"] ["SIGNAL"]]
Generates a one-sample long impulse when the value of "ENTER" input changes from ZERO to something greater.
[inactive_trigger module] => Creates an inactive_trigger.
Note: all usages of module indentical to trigger.
Signals: [["BUSY" "HOLD" "PORTA" "TIME" "LEGATO"] ["KEY" "VELOCITY" "TRIGGER" "INDEX" "DELAY1" "DELAY2"]]
Inactive trigger behaves similar to trigger. However, as it is not active, neither portamento nor trigger delays can be used.
[key_map module] => Creates a frequency mapper, which can be connected to trigger.
[module] => Closes and removes the module.
[module key freq] => Sets the frequency value for a particular key.
[module v1 v2 v3 .... v128] => Sets the entire frequency map.
[module [v1 v2 v3 .... v128]] => Sets the entire frequency map.
[module []] or [module [[]]] => Resets the map to the default value (-8192.0 .. 8192.0).
Note: there is no common behaviour for this type of module.
Signals:none.
Provides key to frequency mapping for triggers (range -8192.0 .. 8192.0). Very good for simulating custom musical scales.
[lfo module] => Creates an lfo module.
Note: all usages of module indentical to trigger.
Signals: [["TRIGGER" "SPEED" "WAVE" "PULSE" "PHASE" "SYNC"] ["SIGNAL" "NEGATIVE" "POSITIVE"]]
LFO generates low frequency oscillations, which can be used as control signals. Very useful for vibrato, tremolo, wah-wah effect and many more. INPUTS:
Trigger: positive slope resets the value if sync is greater than ZERO.
Speed: controls the frequency of LFO. Range between -8192.0 .. 8192.0. Zero represents 1 Hz. Every step of 768 (6 * 128) multiplies speed by 2.
Wave: 0 = sine, 1 = triange, 2 = square, 3 = random, 4 = S/H.
Pulse: pulse width of the signal. Range between -8192.0 .. 8192.0.
Phase: retrigger phase. Range between 0.0 .. 16384.0.
Sync: controlls whether LFO can be retriggered.
OUTPUTS:
Signal: output signal in range between -1.0 .. 1.0.
Negative: output signal in range between -1.0 .. 0.0.
Positive: output signal in range between 0.0 .. 1.0.
[mixer module] => Creates a simple mixer.
Note: module adheres to Common Behaviour.
Signals: [["ENTER"] ["SIGNAL"]]
Mixer copies its input signal to its output. Mixing is achieved by connecting several signals to mixer's input.
[mono_volume module] => Creates a mono volume amplifier module.
Note: module adheres to Common Behaviour.
Signals: [["ENTER" "GATEWAY"] ["SIGNAL"]]
Works identical to volume module, but with monophonic signal.
[oscilloscope module] => Creates an oscilloscope window.
Note: module adheres to Common Behaviour.
Signals: [["SIGNAL"] ["SIGNAL"]]
Displays the input signal. Range of the signal is from -1.0 to 1.0.
[parameter_block module "style" maximum_change] => Creates a parameter block (parameter optional).
Note: module adheres to Common Behaviour.
Additional usage:
[module : *value] => Returns formatted value of the output if style was specified. Otherwise defaults to common behaviour.
Signals: [["SIGNAL"] ["SIGNAL"]]
Parameter block is a box holding a signal value. There are two types of parameter blocks:
Inactive: this is the default type. Its input is basically the same memory location as its output.
Active:, which actively copies its input to its output.
To make an active parameter block it is necessary to pass the maximum change parameter. It can be either integer or double and represents the maximum change of the value per sample. ZERO indicates exact copy (i.e. no ramping effect).
Parameter blocks can have their output values formatted into a text. To activate this feature a style parameter needs to be passed. It can have one of several pre-defined values: "index", "freq", "amp", "time", "onoff", "wave", "ratio" or "fm4algo".
[stereo_amplifier module] => Creates a stereo amplifier module.
Note: module adheres to Common Behaviour.
Signals: [["LEFT" "RIGHT" "GATEWAY"] ["LEFT" "RIGHT"]]
Works similar to amplifer module, but with stereo signals ("LEFT" and "RIGHT"). Good for building linear stereo amplifiers.
[stereo_gateway module] => Creates a stereo gateway module.
Note: module adheres to Common Behaviour.
Signals: [["LEFT" "RIGHT" "GATEWAY"] ["LEFT" "RIGHT"]]
Works similar to gateway module, but with stereo signals ("LEFT" and "RIGHT"). Good for building linear stereo amplifiers.
[stereo_mixer module] => Creates a simple mixer for stereo channels.
Note: module adheres to Common Behaviour.
Signals: [["LEFT" "RIGHT"] ["LEFT" "RIGHT"]]
Mixer copies its inputs signal to its outputs. Mixing is achieved by connecting several signals to mixer's inputs. Note: use DisconnectStereo for connecting stereo modules.
[trigger module] => Creates a trigger.
Note: module adheres to Common Behaviour.
Additional usage:
[module key_map] => Inserts a key-frequency map.
[module []] => Removes key-frequency map.
[module key_map/[] velocity_map/[] => Inserts or removes key-frequency and key-velocity map.
[module keyon key velocity] => Generates keyon signals with velocity.
[module keyon key] => Generates keyon signals with default velocity = 100.
[module keyoff key] => Generates keyoff signals with key.
[module keyoff] => Generates all keys off signals.
Otherwise defaults to common behaviour.
Signals: [["BUSY" "HOLD" "PORTA" "TIME" "LEGATO"] ["KEY" "VELOCITY" "TRIGGER" "INDEX" "DELAY1" "DELAY2"]]
Generates control signals for synthesiser's modules. It can also generate portamento effect.
INPUTS:
Busy: usually receives signal combined from all the relevant ADSR modules. Indicates whether the sound is still audible (1.0) or not (0.0).
Hold: is the input signal from food pedal.
Porta: engages to ramping of the "KEY" output to simulate portamento.
Time: controls the duration of the portamento transition.
Legato: controls the mode of the portamento effect.
OUTPUTS:
Key: is the MIDI key number translated into the range between -8192.0 .. 8192.0.
Velocity: is the MIDI velocity value translated into the range between 0.0 .. 16384.0.
Trigger: translates MIDI keyon/keyoff commands into the range between 0.0 .. 1.0.
Index: is the non-translated MIDI key number.
Delay1: is the "TRIGGER" value delayed by 1 sample.
Delay2: is the "TRIGGER" value delayed by 2 samples.
[velocity_map module] => Creates a velocity mapper, which can be connected to trigger.
[module] => Closes and removes the module.
[module key velocity] => Sets the velocity value for a particular key.
[module v1 v2 v3 .... v128] => Sets the entire velocity map.
[module [v1 v2 v3 .... v128]] => Sets the entire velocity map.
[module []] or [module [[]]] => Resets the map to the default value (0.0 .. 16384.0).
Note: there is no common behaviour for this type of module.
Signals:none.
Provides key to velocity mapping for triggers (range 0.0 .. 16384.0). Very good for simulating velocity response of keyboards.
[volume module] => Creates a stereo volume amplifier module.
Note: module adheres to Common Behaviour.
Signals: [["LEFT" "RIGHT" "GATEWAY"] ["LEFT" "RIGHT"]]
Applies semi-exponential amplification to both input signals. The range of "GATEWAY" input signal is: 0.0 .. 16384.0, where 12800 (100 * 128) is the non-amplified signal.
[wave module "wave-1.wav" "wave-2.wav" ....] => Creates a collection of waves, which can be connected to sampler_operator.
[module] => Closes and removes the module.
Note: there is no common behaviour for this type of module although it still reports empty list of inputs and outputs.
Signals:none.
Creates a collection of waveforms. They can be connected to a sampler operator.
Superstructure
Utilities
[Connect : *modules] => Connects two modules.
This is a shortcut designed to make programs easier to read. It allows for writing [Connect filter "enter" osc "signal"] rather than [filter "enter" osc "signal"].
[ConnectDryWet to *dry *wet] => Connects three stereo modules modules.
This is a shortcut designed to make programs easier to read. It replaces four lines of connections. Works with input modules with "LEFT" and "RIGHT" signals and output modules with "DRYLEFT", "DRYRIGHT", "WETLEFT" and "WETRIGHT" signals.
[ConnectStereo to from] => Connects two stereo modules modules.
This is a shortcut designed to make programs easier to read. It replaces two lines of connections. Works with modules with "LEFT" and "RIGHT" signals.
[DisonnectDryWet to *dry *wet] => Disconnects three stereo modules modules.
This is a shortcut designed to make programs easier to read. It replaces four lines of disconnections. Works with input modules with "LEFT" and "RIGHT" signals and output modules with "DRYLEFT", "DRYRIGHT", "WETLEFT" and "WETRIGHT" signals.
[DisconnectStereo to from] => Disconnects two stereo modules modules.
This is a shortcut designed to make programs easier to read. It replaces two lines of disconnections. Works with modules with "LEFT" and "RIGHT" signals.

[big_keyboard atom callback]
[atom *x *y] => reposition on the screen
[atom] => closes the panel
callback is called as follows:
[callback keyon key 100] => for keyon
[callback keyon key 0] => for keyoff
Creates a panel containing a big keyboard.
[CommandCentre commander CCCB] => Create a command centre.
[commander *x *y] => reposition on the screen.
[commander] => closes the command centre panel.
CommandCentre panel enables effective control on all instruments, which are constructed as superstructures and present on the list of all instruments. It should always be constructed with the predefined callback CCCB. It is possible to use custom callbacks, but its purpose would be rather dubious.
